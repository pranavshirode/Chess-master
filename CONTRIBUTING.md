Contributing

Before you start
- Read docs/STYLE_GUIDE.md for coding conventions.
- Run tests locally (docs/TESTING.md).

How to contribute
1. Fork the repo and create a feature branch: feature/short-description.
2. Make focused commits. Use conventional commit messages (feat:, fix:, chore:, docs:, refactor:).
3. Add/update tests for behavior changes.
4. Open a Pull Request with: description, motivation, testing steps, and linked issue.

Issue reporting
- Provide steps to reproduce, environment, expected vs actual behavior, logs, and screenshots if applicable.

Code review
- PRs will be reviewed for correctness, style, tests, and performance.
