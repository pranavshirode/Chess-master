Style Guide

General
- Keep functions small and focused.
- Prefer pure functions in engine code.
- Add comments for non-obvious chess logic.

Naming
- Use clear names: Board, Move, Piece, generateMoves, applyMove.

Formatting
- Follow project's formatter (prettier, black, clang-format) if present.
- Lint with configured linter (eslint, flake8, etc.).

Testing and docs
- New features must include tests and a brief docs update.
